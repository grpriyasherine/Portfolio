Overview

The Petstore API provides a simple REST interface for managing pet records in a store. This document covers the four core operations on the /pet resource: create, retrieve, update, and delete.

POST
Add a New Pet
Creates a new pet record in the store. Returns the created pet object.
 GET
Find Pet by ID
Retrieves a single pet record using its unique numeric identifier.
 PUT
Update a Pet
Updates an existing pet record. Returns the updated pet object.
 DELETE
Delete a Pet
Permanently removes a pet record from the store by petId.

Base URL

BASE https://petstore.swagger.io/v2

Authentication

ℹ️
The DELETE endpoint accepts an optional api_key header for authentication. All other endpoints in this reference do not require authentication.

POST  /pet

POST
https://petstore.swagger.io/v2/pet
Add a new pet to the store

This endpoint adds a new pet to the pet store. The request body contains the pet details in JSON format. The server returns the created pet object with a confirmed status.

↳ idOptionalInt640Unique identifier of the tag.

HTTP Status Codes

The API uses standard HTTP status codes across all endpoints. The table below lists every code that the /pet resource returns.

Code Status Applies To Description
200 OK POST · GET · PUT · DELETE The request succeeds. The server returns the expected response body.
400 Bad Request GET · PUT · DELETE The request contains an invalid petId. Supply a valid integer.
404 Not Found GET · PUT · DELETE No pet record exists for the given petId. Verify the ID and retry.
405 Method Not Allowed POST · PUT The request body is missing or contains invalid fields.
